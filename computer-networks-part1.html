01 — What is a Network?

The Fundamentals

🏘️
What is a Computer Network?
Explain it like you're 10 years old
▾

Imagine a neighbourhood where every house has a mailbox. If you live alone on an island, you can't send or receive letters. But connect all the houses with roads — now everyone can share things.

💡 A computer network is like a neighbourhood of roads, but for computers. Each computer is a "house," and the network is the roads between them.

Formal definition: Two or more autonomous computing devices connected together so they can exchange information and share resources (printers, files, internet connections).

The keyword is autonomous — each computer can work independently. A network just makes them able to talk to each other.

Why build a network? The honest answer:

Advantage Real-life meaning
Share resourcesOne printer for 20 computers — much cheaper than 20 printers
Share softwareBuy 1 license, install on server, everyone uses it
Share storageAccess files from any machine on the network
CommunicationEmail, chat, video calls — all network-dependent
Cost reductionSharing cuts costs — resource sharing cuts costs
⚠️ Key Disadvantage to remember: Security risk — if the server or central node has a virus, it can spread to all connected computers. Networks are also harder to manage and require specialist staff.

Components of a Network — The 7 Building Blocks

Host / Node
Any computer attached to the network seeking to use or share resources
💻
Every device on a network — your laptop, phone, printer — is a node. When it can be a client or server (i.e., it participates actively), it's called a host.
Server
A host that provides services or resources to other hosts (clients)
🖥️
A server's only job is to serve. File servers share files, print servers manage printers, web servers host websites. A computer becomes a server as soon as it is attached to a network and starts serving.
Client
A host computer that requests services from a server
📱
Your laptop browsing the web is a client — it sends requests ("give me this webpage") to a server. The server responds. This is the client-server model.
NIC (Network Interface Card)
Hardware card giving each device a unique physical identity on the network
🔌
Every NIC has a factory-assigned MAC address — a 6-byte (48-bit) unique number. No two NICs in the world have the same MAC. Also called NIU (Network Interface Unit) or TAP (Terminal Access Point).
Communication Channel
The physical or wireless medium over which data travels
📡
Two types: Wired (twisted-pair, coaxial, optical fiber) and Wireless (radio waves, microwaves, satellites, infrared). Wired = guided media. Wireless = unguided media.
Software (Protocol)
Rules that all parties on a network agree to follow for communication
📋
Without rules, two computers trying to talk would be like two people speaking different languages. A protocol is the agreed-upon language and set of rules. Example: TCP/IP, HTTP, FTP.
Network OS & Services
Software enabling networking tasks: DNS, file sharing, VoIP
⚙️
A Network Operating System is a specialised OS that handles networking tasks. Network services include DNS (Domain Name System — converts website names to IP addresses), file sharing, VoIP (Voice over IP calls like WhatsApp).
02 — Network Types

LAN, WAN, MAN & PAN

Networks are classified by geographical spread — how far they physically extend. Think of it as zoom levels on a map.

🏠
PAN
Personal Area Network
Your personal bubble of devices. Bluetooth connects your phone to your earbuds, smartwatch, and laptop — all within your reach.
Range: ~10 metres
🏢
LAN
Local Area Network
Your school or office building. Computers share one printer, one internet connection. Uses Ethernet cables or WiFi. Fast and cheap to set up.
Range: up to 1 km
🏙️
MAN
Metropolitan Area Network
An entire city. Like broadband internet services connecting multiple offices across a city. Bigger than LAN, smaller than WAN.
Range: 30–40 km (a city)
🌍
WAN
Wide Area Network
Connects cities, countries, continents. LANs connected across large distances form a WAN. The Internet is the world's largest WAN.
Range: Worldwide
📌 Exam Trap: The Internet is a WAN. The Web (World Wide Web) is a part of the Internet — not the whole Internet. Internet = infrastructure. Web = a service running on it.

LAN vs WAN — The Key Differences

FeatureLANWAN

Setup CostCheaperMuch more expensive

Peer-to-Peer vs Client-Server

Peer-to-Peer (P2P)
🤝 "Everyone is equal — I can give you a file, you can give me one."
No dedicated server. Every computer can be both a client and a server. Works well for small networks (up to 10 computers). Easy to set up but hard to secure. BitTorrent is a famous P2P example.
Client-Server
🏦 "There's a bank (server). You go to the bank (client) whenever you need money."
A dedicated server manages everything. Clients request, server responds. More secure, centrally managed. Used in large networks (schools, companies). Also called Master-Slave network.
FeatureClient-ServerP2P Network
SecurityServer controls securityNo central control
ManagementDedicated team neededAnyone can manage
DependencyClients depend on serverNo dependency
PerformanceCan be upgraded (powerful)Slows if machines slow
BackupAll backed up on serverEach user does own backup
03 — The Internet

How the Internet Actually Works

🌐
What is the Internet?
The world's biggest network of networks
▾

Imagine every city has its own internal phone system (a LAN). To call someone in another city, you need to connect city systems together. The Internet is exactly this — it connects millions of smaller networks into one giant super-network.

💡 The Internet is a worldwide network of computer networks — each network runs by a different company/institution, interconnected at "peering points" using a common language: TCP/IP.

How it works:

1. Your data is broken into small pieces called Packets. Each packet is numbered (1, 2, 3...).

2. Packets are sent to the destination's address. They may take different routes.

3. The destination reassembles them in the correct order. Lost packet? Ask for it again.

TCP/IP = the engine: TCP breaks data into packets and reassembles them. IP routes each packet to the correct destination.

InterSpace = futuristic client-server software allowing real-time audio, video, and text chat in dynamic 3D environments.
04 — Switching Techniques

How Data Travels: 3 Methods

Switching = the technique by which data moves from one computer to another across a network. There are 3 fundamentally different approaches.

⚡ Circuit Switching
"Old telephone call — you dial, a dedicated line is reserved just for you for the entire conversation."
A complete physical connection is established before any data is sent. The path is locked and dedicated. If you don't use it, it's wasted.

Example: Traditional telephone network.
📦 Message Switching
"Like passing a baton in a relay race — the message hops from one switching office to the next, waiting at each stop."
The entire message is sent to a switching office, stored temporarily (buffered), then forwarded when a link is free. Also called Store-and-Forward. No size limit on messages. Slower but no dedicated path needed.
📮 Packet Switching
"Like a courier who breaks a heavy parcel into small boxes — each box takes the fastest available route to the destination."
Message is broken into fixed-size packets. Each packet travels independently and may take different routes. Reassembled at destination. Packets stored in RAM (not disk like message switching) → much faster.

This is how the Internet works.
📌 Key difference — Packet vs Message switching: In message switching, data stored on disk (slower). In packet switching, fixed-size packets stored in RAM (faster). Packet switching improves throughput (performance).

Bandwidth

🚗
What is Bandwidth?
The highway analogy
▾
💡 Think of bandwidth as the width of a road. A wider road = more cars at once. Higher bandwidth = more data per second.

Technical definition: The difference between the highest and lowest frequencies a channel can carry. In digital systems: the maximum data speed (bps) the channel supports.

High bandwidth channels → called broadband channels (like fibre optic, coaxial)
Low bandwidth channels → called narrowband channels (like old telephone lines)

A video download needs far more bandwidth than a text email — it has more complex data.

Communication Modes

→
Simplex
One-way only. Like TV broadcast — signals go one direction. Station A sends, Station B only receives.
⇄
Half-Duplex
Both ways but NOT simultaneously. Like a walkie-talkie — one talks while the other listens.
⇆
Full-Duplex
Both ways simultaneously. Like a phone call — both people can speak and hear at the same time.
06 — Transmission Media

The Wires & Waves of Networking

Transmission media = the "roads" data travels on. Two categories: Guided (wired) — physical cables, and Unguided (wireless) — waves through the air.

🔌 Guided (Wired) Media

🔀
Twisted Pair Cable (TPC)
Most common wiring — cheap, flexible, everyday
▾

Two identical copper wires twisted around each other. The twisting is genius — it cancels out crosstalk (when a signal bleeds from one wire to the adjacent wire and corrupts data).

💡 Like two people holding hands and spinning — the spin cancels interference from outside distractions.

UTP (Unshielded Twisted Pair) — cheapest, most common (Ethernet in schools/offices). Max 100m segment. Categories: CAT1→CAT8 (CAT5 = 1Gbps, CAT6 = 10Gbps).

STP (Shielded Twisted Pair) — has a metal foil shield around pairs for extra protection from interference. Better than UTP but heavier and costlier. Max 100m segment.

Disadvantages: High attenuation (signal weakens over distance), needs repeaters, not suitable for broadband (wide bandwidth) applications.

📺
Coaxial Cable
The cable TV cable — better than twisted pair
▾

A solid copper core surrounded by insulation, a copper mesh shield, and an outer plastic jacket. Like an armoured data pipe.

💡 Imagine a garden hose (copper core) wrapped in foam (insulation), then wire mesh (shield) — it's physically shielded against interference.

Thicknet — thicker cable, up to 500m segments.
Thinnet — thinner cable, max 185m segments, cheaper to install.

Advantages: Better data transmission than TPC, can be used for broadband (multiple channels simultaneously, like cable TV), up to 400 Mbps bandwidth.

Disadvantages: More expensive than TPC, not compatible with twisted pair.

💡
Optical Fibre Cable
Light-speed data — the gold standard
▾

Thin strands of glass that carry data as pulses of light. LEDs or laser diodes send the light at one end. A photo-detector receives it at the other.

💡 Like a mirror tube — light bounces off the walls (cladding reflects it back) and travels through the glass core at nearly the speed of light.

3 layers: Core (glass/plastic) → Cladding (reflects light back) → Protective coating (outer jacket).

Single-mode: One path for light. Up to 5 km, 10 Gbps bandwidth. Longer distance.
Multimode: Multiple paths. Up to 1 km, 100 Gbps bandwidth. Higher speeds over shorter distances.

Advantages: Zero electrical/magnetic interference, extremely high bandwidth, secure (nearly impossible to tap — must cut the wire), immune to weather, can carry multiple channels (TV + data simultaneously).

Disadvantages: Very expensive, fragile, difficult to install and splice, most expensive of all cables.

📡 Unguided (Wireless) Media

Microwave (Terrestrial)
Line-of-sight signals between towers — no cables needed for long distances
📡
Works: Parabolic dish antennas on towers send a beam to another tower. Higher tower = further range (up to 100 km between towers). Line-of-sight — must see the next tower.
Pros: No land acquisition cost, works over difficult terrain. Cons: Insecure, affected by rain/storms, limited bandwidth.
Satellite Microwave
Signal bounced off satellites at 22,300 miles above Earth
🛰️
Works: Ground stations transmit signals to satellites (transponders). Satellites amplify and retransmit to other ground stations. Satellites in geostationary orbit (fixed position relative to Earth).
Pros: Huge coverage area, works over oceans. Cons: Very expensive, high atmospheric losses above 30 GHz.
Radio Wave
Radio frequencies — omnidirectional, used for WiFi and mobile
📻
Uses continuous sine waves. Each different radio signal uses a different sine wave frequency. Omnidirectional — spreads in all directions (unlike microwave which is directional). Used in WiFi, walkie-talkies, FM/AM radio. Cons: Insecure, affected by weather.
Infrared
Short-range line-of-sight — your TV remote is infrared
🔴
Uses infrared light. Secure (cannot penetrate walls — can't be intercepted from outside a room). Used in TV remotes, PDAs, garage doors. Limitation: Must have line-of-sight, cannot penetrate walls, short range.
Bluetooth
Short-range wireless for PAN (Personal Area Networks)
📶
Works at 2.400–2.483.5 GHz frequency band. Very short range (up to 30 feet / 10 m). Connects earbuds, keyboards, phones, laptops. Low power radio waves — that's why your battery barely drains. Perfect for PANs.
Laser
Ultra-fast, point-to-point — between buildings
🔆
Much faster than microwave. Point-to-point (unidirectional). Uses a laser transmitter and photo-sensitive receiver at each end. Used between buildings on a campus. Disadvantage: Adversely affected by weather (fog, rain reduces signal).

      IP Address & MAC Address

      Every device on a network needs a unique identity. Two addresses do this — one logical, one physical.

      Address Reference Card

      IPv4 Address (Logical)
      123.101.16.90
      4 bytes (32 bits) · 4 octets · dotted decimal notation · up to 4.3 billion unique addresses
      IPv6 Address (Logical)
      2001:0DC8:E004:0001:0000:0000:0000:F00A
      16 bytes (128 bits) · 8 groups of hex · 340 undecillion unique addresses
      MAC Address (Physical)
      10:B5:03:63:2E:FC
      6 bytes (48 bits) · First 3 bytes = OUI (manufacturer) · Last 3 bytes = card number · Never changes · Assigned by NIC manufacturer
      Key Difference
      IP = Logical (can change)
      MAC = Physical (permanent)
      IP changes when you connect to a different network. MAC is burned into the hardware forever.
